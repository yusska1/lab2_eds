# Лаб №2  
# Электронно-цифровая подпись (ЭЦП)
# Описание
В данной работе реализована электронно-цифровая подпись (ЭЦП) с использованием асимметричного шифрования (RSA) в клиент–серверной архитектуре.

Программа позволяет проверять подлинность сообщения и гарантировать, что оно не было изменено.


# Используемые технологии
- Python
- Flask (сервер)
- requests (клиент)
- cryptography (работа с ЭЦП)


# Структура проекта
- `server.py` — сервер (проверяет подпись и подписывает сообщения)
- `client.py` — клиент (подписывает и проверяет сообщения)


# Как работает ЭЦП
1. Создаётся сообщение  
2. Сообщение подписывается **закрытым ключом**  
3. Подпись проверяется с помощью **открытого ключа**  


# Реализованные сценарии

# 1. Подпись клиента
- клиент подписывает сообщение  
- сервер проверяет подпись  

Результат:
- valid — всё корректно  
- invalid — подпись неверная  


# 2. Подпись сервера
- сервер подписывает сообщение  
- клиент проверяет подпись  

Результат:
- Valid signature — подпись верна
